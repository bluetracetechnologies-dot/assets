# Organization Issue Taxonomy

Use these categories for consistent issue logging across projects:

1. Protocol Compliance
- Incorrect interpretation of standard behavior, frame format, timing, or addressing.

2. Hardware Behavior
- Board-specific electrical/runtime behavior not reflected in documentation.

3. Verification Traceability
- Claims made without linked instrument settings, sample IDs, or test evidence.

4. Diagram Quality
- Layout overlap, unreadable typography, inconsistent legend, broken exports.

5. Encoding and Formatting
- Corrupted characters, unit symbol issues, inconsistent notation.

6. Branding and Metadata
- Missing or inconsistent logo usage, document footer mismatch, stale revision/date.

7. Process and Handoff
- Missing session summary, lost context between tools, inconsistent update procedure.

## Required incident fields
- Date
- Category
- Project
- Impact
- Root cause
- Fix
- Prevention
