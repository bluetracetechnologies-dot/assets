# Technical Requirements Checklist

## Architecture and Existing Assets
- Confirm target platform and revision for ESP32-S3
- Confirm available schematic, BOM, and PCB/PCBA revisions
- Confirm provided reference schematic and block diagram version
- Confirm mechanical envelope and connector constraints

## Interfaces and Features
- Clarify required implementation for reference board adaptation, schematic update, PCB layout changes, audio path integrity
- Identify mandatory interfaces and pin-mux constraints
- Define audio path requirements (mic, codec, amp, speaker path)
- Define Wi-Fi antenna placement and RF keep-out constraints
- Define compatibility constraints if any

## Validation and Acceptance
- Define design-review completion criteria
- Confirm test setup availability and access model
- Confirm expected evidence format (annotated schematic/PCB diffs and review notes)
- Confirm prototype bring-up and validation ownership split

## Delivery and Release
- Confirm design tools and output formats required by client
- Confirm handover artifacts: schematic, PCB, BOM, fabrication outputs
- Confirm post-delivery support window for revision feedback
- Confirm final signoff process
